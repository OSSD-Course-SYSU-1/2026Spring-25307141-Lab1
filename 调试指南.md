# 调试指南 - 聊天功能问题排查

## 🔍 问题排查步骤

我已经添加了详细的调试日志，现在请按照以下步骤排查问题：

### 步骤1：运行应用并查看日志

1. 在 DevEco Studio 中运行应用
2. 打开 **HiLog** 日志窗口（底部工具栏）
3. 在日志过滤框中输入：`ChatPage|Contact|ChatDataManager`

### 步骤2：测试点击联系人

1. 点击底部"通讯录"标签
2. 点击任意联系人（如"ArdWard"）

### 步骤3：查看日志输出

#### 正常情况下的日志输出：

```
Contact: 点击了联系人: ArdWard
Contact: 准备跳转到聊天页面，用户名: ArdWard
Contact: 跳转成功
ChatPage: aboutToAppear 开始执行
ChatPage: router.getParams() 结果: {"name":"ArdWard"}
ChatPage: 接收到的用户名: ArdWard
ChatPage: 准备获取聊天记录
ChatDataManager: 获取聊天记录，用户名: ArdWard
ChatDataManager: 找到聊天记录，数量: 3
ChatPage: 获取到的聊天记录数量: 3
ChatPage: 聊天记录内容: [...]
ChatPage: dataList已更新，长度: 3
ChatPage: aboutToAppear 执行完成
```

### 步骤4：根据日志判断问题

#### 情况A：看不到 "Contact: 点击了联系人" 日志
**问题**：点击事件没有触发
**可能原因**：
- ListItem被其他组件遮挡
- onClick事件被拦截

**解决方案**：检查UI层级结构

#### 情况B：看到 "Contact: 点击了联系人" 但没有 "准备跳转"
**问题**：联系人被识别为特殊项
**检查**：确认点击的不是"新的朋友"、"群聊"等特殊项

#### 情况C：看到 "准备跳转" 但没有 "跳转成功"
**问题**：路由跳转失败
**可能原因**：
- 页面路径错误
- 页面未注册

**检查**：查看是否有 "跳转失败" 日志

#### 情况D：看到 "跳转成功" 但没有 "aboutToAppear 开始执行"
**问题**：ChatPage页面没有加载
**可能原因**：
- 页面路径错误
- 页面组件有问题

#### 情况E：看到 "接收到的用户名: undefined"
**问题**：参数没有正确传递
**检查**：查看 "router.getParams() 结果" 日志

#### 情况F：看到 "获取到的聊天记录数量: 0"
**问题**：ChatDataManager没有找到聊天记录
**检查**：查看 "ChatDataManager: 未找到聊天记录" 日志

#### 情况G：看到 "dataList已更新，长度: 3" 但页面空白
**问题**：UI没有正确渲染
**可能原因**：
- ForEach没有正确更新
- List组件有问题

## 🐛 常见问题及解决方案

### 问题1：点击后没反应
**检查**：是否有 "Contact: 点击了联系人" 日志
**如果没有**：点击事件被拦截，检查UI层级
**如果有**：继续查看后续日志

### 问题2：跳转后页面空白
**检查**：是否有 "dataList已更新，长度: X" 日志
**如果有**：UI渲染问题，检查List和ForEach
**如果没有**：数据加载问题，检查前面的日志

### 问题3：显示错误的聊天记录
**检查**：查看 "接收到的用户名" 是否正确
**如果不正确**：参数传递问题
**如果正确**：ChatDataManager数据问题
