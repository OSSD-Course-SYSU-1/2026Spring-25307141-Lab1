# 响应式聊天列表实现说明

## 一、实现概述

本项目实现了一个完整的响应式聊天列表UI，支持手机与平板的自适应布局，类似微信的聊天列表界面。

### 核心特性

✅ **一套代码多端适配**：同一套代码同时适配手机端（窄屏）与平板端（宽屏）
✅ **自动布局切换**：根据屏幕宽度自动切换布局结构
✅ **横竖屏自适应**：支持横竖屏切换，布局自动调整
✅ **UI不变形、不溢出、不重叠**：所有样式支持自适配

---

## 二、屏幕分级策略

### 断点定义

| 设备类型 | 屏幕宽度 | 布局方式 | 说明 |
|---------|---------|---------|------|
| 手机 | < 600vp | 单列布局 | 点击列表项跳转详情页 |
| 小平板 | 600vp - 840vp | 双列布局 | 左侧列表（30%）+ 右侧详情（70%） |
| 大平板 | > 840vp | 优化分栏布局 | 列表固定宽度280-400vp |

### 布局判断逻辑

```typescript
private updateLayoutMode(): void {
  const width = this.screenWidth;
  
  // 只有平板横屏或大屏设备才使用分栏布局
  this.useSplitLayout = 
    (this.deviceType === DeviceType.TABLET && this.isLandscape) ||
    (this.isLargeScreen && width >= 600);
}
```

## 四、核心实现细节

### 1. 响应式布局实现

#### 使用 `@StorageLink` 连接全局设备状态

```typescript
@StorageLink('deviceType') deviceType: DeviceType = DeviceType.PHONE;
@StorageLink('isLandscape') isLandscape: boolean = false;
@StorageLink('screenWidth') screenWidth: number = 360;
@StorageLink('fontSizeScale') fontSizeScale: number = 1.0;
@StorageLink('paddingScale') paddingScale: number = 1.0;
```

#### 自适应字体大小

```typescript
private getAdaptiveFontSize(baseSize: number): number {
  return baseSize * this.fontSizeScale;
}
```

#### 自适应间距

```typescript
private getAdaptivePadding(basePadding: number): number {
  return basePadding * this.paddingScale;
}
```

### 2. Master-Detail 模式

#### 平板端分栏布局

```typescript
if (this.useSplitLayout) {
  // 平板分栏布局（Master-Detail模式）
  this.buildSplitLayout();
} else {
  // 手机单列布局
  this.buildSingleLayout();
}
```

#### 左侧列表（30%宽度）

```typescript
Column() {
  ResponsiveTopBar({ title: '微信(5)' });
  ResponsiveChatList({
    chatList: $chatList,
    selectedChatId: $selectedChatId,
    onChatItemClick: (chatId: string) => {
      this.handleChatItemClick(chatId);
    }
  });
}
.width('30%')
```

#### 右侧详情（70%宽度）

```typescript
ResponsiveChatDetail({
  conversation: this.currentConversation,
  showBackButton: false
});
```

### 3. 交互逻辑

#### 手机端：点击跳转详情页

```typescript
if (!this.useSplitLayout) {
  router.pushUrl({
    url: 'pages/chat/ChatPage',
    params: { name: chatItem.nickname }
  });
}
```

#### 平板端：实时更新右侧内容

```typescript
if (this.useSplitLayout) {
  this.selectedChatId = chatId;
  this.loadConversation(chatId, chatItem.nickname);
}
```

### 4. 滑动删除功能

使用 `SwipeDeleteItem` 组件包装聊天项：

```typescript
SwipeDeleteItem({
  itemId: item.id,
  onDelete: () => {
    this.deleteChatItem(index, item.nickname);
  }

  ResponsiveChatItem({
    chatData: item,
    isSelected: this.selectedChatId === item.id

```

## 五、样式自适配

### 1. 头像大小适配

| 设备 | 头像大小 |
|------|---------|
| 手机 | 60vp |
| 平板 | 70vp |

```typescript
private getAvatarSize(): number {
  if (this.deviceType === DeviceType.TABLET) {
    return 70 * this.paddingScale;
  }
  return 60 * this.paddingScale;
}
```

### 2. 列表项高度适配

| 设备 | 列表项高度 |
|------|-----------|
| 手机 | 80vp |
| 平板 | 90vp |

```typescript
private getItemHeight(): number {
  if (this.deviceType === DeviceType.TABLET) {
    return 90 * this.paddingScale;
  }
  return 80 * this.paddingScale;
}
```

### 3. 分隔线适配

分隔线起始边距根据头像大小动态计算：

```typescript
private getDividerStartMargin(): number {
  if (this.deviceType === DeviceType.TABLET) {
    return 105 * this.paddingScale; // 平板：头像70 + 间距35
  }
  return 90 * this.paddingScale; // 手机：头像60 + 间距30
}
```

## 七、状态管理

### 1. 数据模型

```typescript
export interface ChatItemModel {
  id: string;                    // 唯一标识
  headSrc?: Resource | string;   // 头像资源
  nickname: string;              // 昵称/群名
  content: string;               // 最近消息预览
  time: string;                  // 时间

  isPinned?: boolean;            // 是否置顶
```

## 十一、注意事项

1. **不要硬编码屏幕尺寸**：所有尺寸使用 `getAdaptiveFontSize()` 和 `getAdaptivePadding()` 方法
2. **使用 layoutWeight 而非固定宽度**：让布局自动分配空间
3. **测试多种设备**：确保在不同设备上显示正常
4. **性能优化**：使用懒加载和列表优化
5. **状态管理清晰**：使用 @State / @Link / @StorageLink 管理状态
