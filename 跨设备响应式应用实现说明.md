# 跨设备响应式应用实现说明

## 📋 实现清单

### ✅ 核心功能实现

1. **设备自动适配**
   - ✅ 自动识别设备类型（手机 / 平板 / 折叠屏扩展态）
   - ✅ 根据窗口宽度动态切换布局
   - ✅ 不同设备下 UI 结构自动重组，而不是简单缩放

2. **跨设备流转**
   - ✅ 实现类似 HarmonyOS 的 Continuation / 多端协同体验
   - ✅ 应用状态可以"无感恢复"
   - ✅ 页面内容、滚动位置、选中项保持一致

3. **布局切换规则**
   - ✅ 手机模式（< 600vp）：单列布局，列表页 → 详情页（跳转）
   - ✅ 平板模式（≥ 600vp）：双栏 Master-Detail 布局
   - ✅ 动态切换：自动从"页面跳转模式"切换为"双栏模式"

4. **状态同步机制**
   - ✅ 全局状态管理方案（AppStateStore）
   - ✅ 状态持久化（使用 AppStorage）
   - ✅ 状态在设备切换时自动恢复

## 🎯 核心实现

### 1. 全局状态管理（AppStateStore）

#### 功能特性
- 单例模式，全局唯一实例
- 管理应用全局状态（选中项、滚动位置、输入内容等）
- 支持状态持久化（使用 AppStorage）
- 支持跨设备状态同步
- 状态变更监听机制

#### 状态内容
```typescript
interface AppState {
  // 设备相关
  deviceType: DeviceType;           // 设备类型
  layoutMode: LayoutMode;           // 布局模式
  windowWidth: number;              // 窗口宽度
  windowHeight: number;             // 窗口高度
  isLandscape: boolean;             // 是否横屏
  
  // 选中状态
  selectedItemId: string;           // 选中的列表项ID
  selectedDetail: DetailModel | null; // 选中的详情数据
  
  // 滚动位置
  listScrollPosition: ScrollPosition;  // 列表滚动位置
  detailScrollPosition: ScrollPosition; // 详情滚动位置
  
  // 输入状态
  currentInputValue: string;        // 当前输入内容
  
  // 页面路径
  currentPath: string;              // 当前页面路径
  navigationStack: string[];        // 导航栈
  
  // 临时数据
  tempData: Map<string, Object>;    // 临时数据存储
}
```

#### 使用示例
```typescript
// 获取状态存储实例
const stateStore = AppStateStore.getInstance();

// 更新选中项
stateStore.updateSelectedItem('item-123');

// 更新滚动位置
stateStore.updateScrollPosition('list', { x: 0, y: 100 });

// 更新输入内容
stateStore.updateInputValue('Hello World');

// 导出状态（用于跨设备流转）
const stateJson = stateStore.exportState();

// 导入状态（用于跨设备流转）
stateStore.importState(stateJson);
```

## 🔄 布局切换规则

### 手机模式（< 600vp）
- 单列布局
- 页面结构：列表页 → 详情页（跳转）
- 使用 Stack / Navigation 结构

### 平板模式（≥ 600vp）
- 双栏 Master-Detail 布局
- 左侧：列表（30%宽度）
- 右侧：详情（70%宽度）
- 点击列表不跳转页面，在右侧更新内容

### 动态切换要求
- 从手机切到平板：
  - 自动从"页面跳转模式"切换为"双栏模式"
  - 当前选中的 item 自动加载到右侧详情
- 从平板切回手机：
  - 自动退化为列表+详情分离页面
  - 保持当前选中状态

## 🚀 跨设备流转逻辑

### 1. 设备切换检测
- 使用 `window.getWindowInfo()` 监听窗口尺寸变化
- 判断 breakpoint：phone / tablet / foldable

### 2. UI重构策略
- 不刷新应用，仅重建 layout tree
- 使用条件渲染：
  ```typescript
  if (isTablet) {
    renderMasterDetail()
  } else {
    renderNavigationFlow()
  }
  ```

### 3. 状态恢复机制
- 切换设备后：
  - 自动恢复 selectedItemId
  - 自动滚动到上次位置
  - 自动进入对应详情页或右侧面板
